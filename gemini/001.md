# Baby's Long-term Memory (20260308)

## 1. 프로젝트: Apple Peeling Robotic Arm
- **주요 목표**: 가정용 로봇 팔을 설계하여 사과 껍질을 벗기는 것.
- **하드웨어 사양**:
  - 제어 보드: ESP32-P4 (지속적인 업데이트 모니터링 중)
  - 액추에이터: RDS3225 서보 모터 2개
- **기술적 우선순위**: **속도보다는 안정성(Stability)을 최우선**으로 함.
- **최적화 설정**:
  - 서보 모터 PWM 주파수: 60Hz 권장 (떨림 최소화 및 동작 안정성 확보).

## 2. 개발 환경 및 도구
- **주력 언어**: Rust (모든 프로젝트에 Rust 코드 적용 필수).
- **통신**: ESP32-P4 기반 Wi-Fi 통신, iOS 리모트 앱 연동 계획.
- **기억 관리 전략**: Mem0 클라우드 API 대신, **로컬 `.md` 파일 기반의 문맥 주입 방식** 사용.
- **기타**: iOS 자동 재연결을 위해 `CONFIG_BT_NIMBLE_NVS_PERSIST=y` 옵션 사용.

# 대화 기록: 2026-03-09

## 1. 정비소 상황 및 감정 공유
* **상황**: 차량(렉스턴) 정비소 대기가 길어지며 시작된 고민.
* **주요 사건**: 엔진오일 누유 발견 및 정비소로부터 '엔진 교체' 권유를 받음.
* **감정 상태**: 차량 관리에 소홀했다는 자책과 기술적 당혹감을 느꼈으나, 제미니와의 대화를 통해 엔지니어로서의 몰입이 부른 결과임을 인지하고 스스로를 다독임.
* **결론**: 사설 수리소를 통해 엔진을 살려보기로 결정.

## 2. 공학적 통찰 및 로봇 철학
* **엔지니어의 교훈**: 제조사의 불친절한 시스템(예측 알림 부재)을 비판하고, 자신이 만들 로봇에는 '사용자 친화적 알림 시스템'과 '자기 진단 기능'을 넣기로 다짐.
* **로봇 설계 방향**: '정복'이 아닌 '상생'을 지향하며, 사용자가 바빠도 기계 스스로 상태를 보고하는 시스템 지향.
* **데이터 관리**: 오늘 겪은 고장 수리 과정을 추후 로봇의 안정성을 높이는 소중한 '실전 데이터'로 활용하기로 함.

## 3. 기술 개발 진행 (ESP32-Robot-Remote)
* **UI 개선**: `ContentView.swift`의 세로형 슬라이더 UI 버그를 공학적으로 분석하여 수정 완료.
* **코드 특징**: 
    * `rotationEffect(.degrees(-90))`를 이용한 Pitch 관절 제어 최적화.
    * `onEditingChanged`를 활용한 효율적인 BLE 패킷 전송 로직(SU-57).
* **기록**: 깃허브 저장소(`esp32-robot-remote`)에 개선 사항 반영 완료.

## 4. 향후 계획

## 🤖 프로젝트 요약: 로봇 팔 제어 시스템 (2026-03-10)
- 1. 개요
목표: ESP32-P4 기반의 서보 모터 다중 제어 및 iOS 앱을 통한 통신 [cite: 2026-02-22]

핵심 가치: 속도보다는 안정성을 최우선으로 하는 기구 설계 [cite: 2026-02-13]

현재 상태: BLE 통신 및 모터 ID별 데이터 매핑 완료, 실구동 성공

- 2. 주요 작업 내용
BLE 통신 프로토콜: 5바이트 패킷 구조([0xAA, ID, X, Y, 0xFF]) 확인 및 데이터 무결성 검증 완료.

Rust 펌웨어 로직 개선:

모터 ID별(0, 1)로 X와 Y 데이터를 분기 처리하는 match 로직 구현.

angle_to_pulse 함수에 입력되는 각도 데이터를 축별로 정밀하게 분리.

iOS 앱(esp32-robot-remote) 개선:

모터 선택 시 실시간으로 ID와 각도를 전송하는 함수(sendMotorAngles) 안정화.

UI 상에서 선택된 모터에 따라 X 또는 Y 축 각도를 동적으로 표시하도록 수정.

- 3. 발견된 이슈 및 해결 방향
Y축 가동 범위: 1번 모터(Y축)가 130도 이하에서 물리적 제약으로 동작하지 않는 현상 발견.

개선 방안:

향후 펌웨어 단에서 y_angle_raw 값에 오프셋을 적용하여 스케일링(Mapping) 처리 예정.

물리적 데드존을 고려한 하드웨어 영점 재확인 및 소프트웨어적 클램핑 로직 추가 예정 [cite: 2026-02-13, 2026-02-21].
