# 프로젝트 로그: ESP32 로봇 팔 오케스트레이션 (사과 깎기 프로젝트)

## 1. 프로젝트 핵심 철학 (Core Philosophy)

- **안정성 우선 (Stability Over Speed)**: 속도보다 로봇의 정밀한 동작과 하드웨어의 장기적인 안정성을 최우선으로 한다.
- **지속 가능성**: 잦은 고장이나 과열 없이 10개월 이상 무중단으로 동작하는 시스템 수준의 견고함을 지향한다.
- **인간 중심**: 로봇이 인간과 평화롭게 공존하며, 장애인을 포함한 모든 이에게 도움을 줄 수 있는 가치를 추구한다.

## 2. 하드웨어 스펙 및 환경

- **모터**: RDS3225 서보 모터 4축 (메탈 기어 사용)
- **컨트롤러**: ESP32-P4 및 ESP32-C6 (핀맵 설정 완료)
- **통신**: BLE를 통한 무선 제어 및 오케스트레이션
- **언어**: Rust (소유권 및 안전성을 통한 메모리 관리)
- **전원**: 안정적인 전원 공급 및 커패시터를 활용한 지터(Jitter) 제어

## 3. 기술적 로드맵 (Technical Roadmap)

- **모션 프로파일**: 급격한 가속을 지양하고 S-Curve 알고리즘을 통한 부드러운 가속/감속 제어.
- **오케스트레이션**: 각 관절의 독립적인 움직임이 아닌, 유기적인 결합(Motion Blending)을 통한 우아한 궤적 생성.
- **AI 통합**: Gemini CLI 및 MCP(Model Context Protocol)를 활용하여 터미널 내에서 실시간 로봇 상태 모니터링 및 제어 로직 최적화.

# 🤖 로봇 팔 제어 시스템 기술 문서 (RDS3225 이식 프로젝트)

## 1. 프로젝트 개요

- **기술 스택**: Rust (임베디드), Swift (앱), BLE (무선 제어) [cite: 2026-03-04].

## 2. 수학적 기반 (Kinematics & Control)

### 🔹 역기구학 (Inverse Kinematics)

로봇 팔의 손끝 좌표 (x, y)를 기반으로 각 관절의 각도 θ1, θ2를 계산하여 모터를 제어함 [cite: 2026-03-17].

- **코사인 법칙**: cos(θ2) = (x^2 + y^2 - l1^2 - l2^2) / (2 _ l1 _ l2) [cite: 2026-03-17]
- **각도 계산**: θ1 = atan2(y, x) - atan2(l2 _ sin(θ2), l1 + l2 _ cos(θ2)) [cite: 2026-03-17]

### 🔹 안정성 가드 (Safety Logic)

물리적 한계를 넘어선 동작을 방지하기 위해 Rust의 타입 시스템과 가드 로직을 활용함 [cite: 2026-03-17].

- **팔꿈치 설정**: `ElbowConfig::Up` 또는 `ElbowConfig::Down`을 명시하여 동작 의도 명확화 [cite: 2026-03-17].
- **범위 보호**: 도달 불가능한 좌표 입력 시 `None`을 반환하여 모터 과부하 및 오작동 방지 [cite: 2026-03-17].

## 3. Rust 구현 코드 패턴

```rust
pub enum ElbowConfig {
    Up,
    Down,
}

pub struct JointAngles {
    pub theta1: f32,
    pub theta2: f32,
}

pub fn calculate_ik(x: f32, y: f32, l1: f32, l2: f32, config: ElbowConfig) -> Option<JointAngles> {
    let dist_sq = x * x + y * y;
    // 도달 가능성 검증 (안정성 가드)
    if dist_sq > (l1 + l2).powi(2) || dist_sq < (l1 - l2).powi(2) {
        return None;
    }
    // ... (상세 계산 로직)
    Some(JointAngles { theta1, theta2 })
}
```

# 🤖 초보자를 위한 AI 핵심 개념 요약

## 1. AI(인공지능)란?

- **정의**: 기계가 인간처럼 생각하거나 행동하도록 만드는 기술의 총칭입니다.
- **핵심**: 예전에는 사람이 일일이 규칙을 적어줬다면(예: "모터 각도가 90도면 멈춰라"), 지금의 AI는 **데이터를 통해 스스로 패턴을 학습**합니다.

## 2. LLM(Large Language Model, 예: 제미니)

- **정의**: 엄청나게 많은 글(데이터)을 학습하여, 다음 단어를 예측하는 '확률적 뇌'입니다.
- **특징**: 단순히 지식을 뽐내는 것이 아니라, 언어의 문맥을 이해하고 프로그래밍 코드를 짜거나 수학 문제를 푸는 등 '추론'이 가능합니다.

## 3. AGI(범용 인공지능)와 지금의 AI 차이

- **현재의 AI(Narrow AI)**: 사과 깎기, 번역, 이미지 생성 등 **'특정 작업'**을 아주 잘 수행합니다.
- **AGI(범용 AI)**: 인간이 할 수 있는 모든 지적인 작업을 수행할 수 있는 '완전한 지능'입니다. 아직은 도달하지 못한 미래의 목표입니다.

## 4. 왜 지금의 AI가 로봇 제어에 도움을 줄까?

- **번역기 역할**: 복잡한 수학 공식(기구학)을 프로그래밍 언어(Rust)로 **번역**해줍니다.
- **파트너**: 모터가 고장 났을 때 어떤 부품이 더 나은지, 왜 그런지를 함께 논의하는 **지적인 토론 상대**가 되어줍니다.
